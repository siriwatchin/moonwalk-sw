Moon Walk · Metrics Explainer

What we show, and how we compute it

Every Moon Walk number traces back to two raw signals. This page shows, for each metric, the input signals → the calculation → the output — with live calculators you can tweak. The flagship WSFC loop lives on the loading metrics (§3) and a Whoop-Strain-style composite (§4).

Tier 1 reliable Tier 2 trend-only Tier 3 relative loading Tier 4 derived/composite LIVE (IMU-only, buildable now) AVAILABLE (Handle Load sensor built & validated, ADR-0010)

0The two raw signals everything is built from

IMU — LSM9DS1 (in hand)
6-axis, ~100–200 Hz. Gives the swing-axis angular rate (gyro) and acceleration. From these: plant timestamps (rate→0 + impact spike), swing angle θ (gyro integral), L/R step timing, stillness windows.
Handle Load — LPS22HB + bladder (designed)
Barometer reads trapped-air pressure under the grip. Tared at session start, mapped pressure→load by a fitted polynomial, expressed as % of the patient's own baseline — never Newtons, never %body-weight.

1Temporal metrics — reliable, IMU-only (live today)

Stick Cycle TimeTier 1live
IMU: plant timestamps→consecutive-plant interval→cycle time (s)
cycle_time = tplant[n] − tplant[n−1]

A plant is detected when angular rate crosses ~0 with an acceleration impact spike (a hysteresis crossing in the CycleDetector). One plant-to-plant interval = one Stick Cycle — a proxy for the gait cycle, not identical to it.

CadenceTier 1live
cycle time→invert & scale to /min→cadence (cyc/min)
cadence = 60000 / cycle_time_ms  // exactly what realtime/hub.py computes, rolling-averaged

Stick Duty FactorTier 1live
IMU: stillness window→÷ cycle time→planted fraction
duty_factor = planted_duration / cycle_time

Fraction of the cycle the cane is planted (inferred from IMU stillness — crisper once the Handle Load sensor confirms "loaded"). Not leg stance time.

2Spatial metrics — trend-only (IMU, never absolute)

Stride Length & Gait VelocityTier 2trend-only
gyro swing angle θ→pendulum model→stride→× cadence→velocity
stride ≈ L · sin(θ)  (L = stick length, ZUPT-bounded)   velocity = stride · cadence

Presented only as change-over-time vs the user's own baseline — never as a clinical absolute. Requires one-time stick-length calibration; not yet validated against ground truth.

3Loading metrics — the WSFC's headline signals (Handle Load sensor built & validated, ADR-0010)

Handle Load (relative)Tier 3available
barometer pressure P→− session tare→polynomial cal→% of baseline
load = a·(P−Ptare)² + b·(P−Ptare) + c  (quadratic fit, Marquardt 2022 form) load% = 100 · load / baseline_load

Expressed as % of the patient's own measured baseline cane-dependence. kgf is bench-calibration / clinician readout only — never a marketed force or %body-weight.

Sensor evidence: barometer-in-elastomer force sensors Marquardt 2022, Cerveri 2017; drift-stable bubble sensor Dabling 2012.

4Derived / composite metrics — the "score" layer

Higher-order numbers built from the metrics above — a gait-quality score (live today) and a Whoop-Strain-style session dose (now available — load sensor built & validated). All self-referenced to the patient's own baseline; never a population norm, never fall-risk.

Step Rhythm & SymmetryTier 4 derivedlive · IMU-only
L/R step times→symmetry ratio + step-time CV→0–100 quality score
SR = min(tpar, tnonpar) / max(tpar, tnonpar)  (1.0 = perfectly symmetric) consistency = 1 − CV  CV = SD(step_time) / mean(step_time) score = 100 · (0.6·SR + 0.4·consistency)

CV is surfaced as "step-timing consistency / steadiness" — never as fall risk (a banned framing). Trends toward SR = 1.0 and lower CV vs the patient's own baseline.

Symmetry ratio standardization: Patterson 2010, Gait & Posture (707 cites); variability poststroke: Chisholm 2014; symmetry responds to therapy: Yoshioka 2022.

Load-dose structure: García-Ramos 2015 (TRIMP), Tibana 2018 (session-RPE); rehab loading dose: Mercer 2009, Ribeiro 2019; Whoop signals (not Strain) validated: Miller 2022.
